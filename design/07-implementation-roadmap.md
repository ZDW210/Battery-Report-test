# 实施路线

## Phase 0：准备阶段

目标：搭好 RuoYi 基座，不迁移业务。

- 拉取 RuoYi-Vue-Pro
- 本地启动后端、管理端、数据库、Redis
- 确认数据库类型：MySQL 或 PostgreSQL
- 确认是否启用租户
- 确认小程序继续独立维护

交付物：

- RuoYi 基础环境可运行
- 管理端可登录
- 数据库初始化完成

## Phase 1：业务主数据

目标：完成设备、客户、项目的基础管理。

- 新增 `yudao-module-energy`
- 新增客户表、项目表、设备表
- 生成基础 CRUD
- 管理端新增菜单
- 导入现有设备和客户数据

交付物：

- 设备台账
- 客户管理
- 项目/场站管理

## Phase 2：数据接入和实时状态

目标：打通安科瑞 EIOT 数据链路（入向）。

- 新增 EIOT 实时数据接口
- 新增 EIOT 告警接口
- 写入设备最新状态
- 写入采集明细
- 写入同步日志
- 保留原始报文归档

交付物：

- 设备实时状态
- 设备历史曲线
- EIOT 同步日志

## Phase 2.5：EIOT 反向控制

目标：实现本系统主动调用 EIOT 远程操控能力。

- 新增 EIOT 凭证管理（`energy_eiot_credential`）
- 实现 AES-128-CBC 加密登录和 Token 管理
- 封装 SWITCH / REFRESH / FORCESWITCH 控制指令
- 新增操控日志（`energy_device_control_log`）
- 管理端设备操控页面
- 小程序设备操控（SWITCH 仅限已授权设备）

交付物：

- EIOT 凭证配置
- 设备手动合闸/分闸
- 操控日志查询

## Phase 3：告警闭环

目标：告警从产生到处理形成闭环。

- 告警入库
- 告警确认
- 告警关闭
- 告警统计
- 小程序告警查看和确认
- 微信或短信通知预留

交付物：

- 管理端告警中心
- 小程序告警中心

## Phase 4：计费和充放电记录

目标：完成商业闭环，充放电会话与 EIOT 控制联动。

- 计费规则 CRUD
- 充放电会话状态机（PENDING → CHARGING/DISCHARGING → COMPLETED/ABNORMAL → SETTLED）
- 会话启停自动调用 EIOT SWITCH
- 电能快照（start_epi/start_epe → end_epi/end_epe）
- 自动费用计算
- 账单查询
- 导出报表

交付物：

- 计费规则管理
- 充放电会话管理（含手动启停）
- 小程序账单

## Phase 5：调度和运维

目标：完成设备运营能力。

- 调度任务
- 调度策略
- 工单管理
- 巡检记录
- 异常处理流程

交付物：

- 调度管理
- 运维工单

## 迁移建议

你当前本地已有 NestJS + Prisma 后端、uni-app 小程序、worker。建议迁移顺序如下：

1. 小程序暂时不动，只调整接口地址和数据结构。
2. worker 暂时保留，继续负责 EIOT 接收和归档。
3. 先把 Prisma 模型迁移成 RuoYi 的业务表。
4. 设备、告警、客户、计费、充放电记录逐个迁移。
5. 等 RuoYi 后端稳定后，再决定是否下线 NestJS 后端。

## 第一版最小可用范围

MVP 建议只做以下内容：

- 管理端登录和权限
- 设备台账
- 设备实时状态
- 告警列表和确认
- 客户管理
- 充放电记录
- 小程序设备列表、设备详情、告警列表
- EIOT 实时数据和告警接入

这样可以最快验证业务闭环，再继续扩展调度、账单、工单和报表。
