# 管理端设计

## 菜单结构

```text
移动储能
├── 首页
├── 数据面板
├── 报表面板
├── 数据报表
├── 设备管理
├── 项目管理
├── 客户管理
├── 客户账号管理
├── 计费规则
└── 其他
    ├── 报警信息
    ├── 车辆管理
    ├── 充放电任务
    ├── 小程序用户
    ├── 用户授权
    ├── 扫码刷卡记录
    └── EIOT 同步日志
```

侧边栏中“其他”为折叠菜单，放在“客户账号管理”和“计费规则”之后；低频运维、授权、车辆、日志类功能统一收纳在该父级下，减少主菜单长度。

## 首页运营面板

### 关键指标

- 设备总数
- 在线设备
- 故障设备
- 今日充电量
- 今日放电量
- 今日收益
- 未确认告警
- 当前总功率

### 图表

- 设备状态分布
- 24 小时功率曲线
- 7 日电量趋势
- 7 日告警趋势
- 客户/项目收益排行

### 数据面板

- 放在运营面板下方，作为接收数据的快速图表入口。
- 支持选择电表、时间范围和数据类型。
- 数据类型按当前 EIOT 已接收字段分组：有功功率、相电压、相电流、功率因数、有功电能。
- 图表复用遥测曲线接口，只展示真实入库数据，不补造未接收字段。
- 面板保留跳转到实时监控详细数据页的入口；导出、逐日极值和固定时间轴明细仍在实时监控的数据查看板块完成。

- 查看告警

### 表单必填

- 设备编号、设备名称、设备类型、设备状态
- 网关序列号、电表序列号、仪表编号
- 所属客户、项目场站

### 台账约束

- 设备台账必须能通过 `meterNo` 或 `gatewaySn + meterSn` 匹配 EIOT 实时报文、告警报文、扫码二维码和刷卡上报。
- 列表必须支持按设备编号、设备名称、网关序列号、电表序列号、仪表编号查询。
- 台账状态用于提示现场资料是否补齐：所属客户、项目场站、网关序列号、电表序列号、仪表编号均存在时显示“完整”，否则显示“待补全”。

## 车辆绑定

### 列表字段

- 车辆编号
- 车牌号
- 二维码编码
- 绑定设备
- 客户
- 项目/场站
- 状态
- 更新时间
- 操作

### 操作

- 新增车辆绑定
- 编辑车辆绑定
- 停用/启用车辆
- 更换绑定设备
- 生成或复制二维码内容

### 约束

- 车辆绑定不再作为新扫码放电主流程的必需步骤，仅作为可选车辆资产绑定和旧二维码兼容。
- 新二维码内容优先承载设备/充电桩/仪表识别信息，例如 `deviceNo`、`deviceId`、`meterNo` 或 `gatewaySn + meterSn`。
- 旧车辆二维码仍可兼容 `qrCode`、`vehicleNo`、`plateNo` 或 `vehicleId`；停用车辆兼容码不能通过小程序扫码校验。

## 客户账号权限

该板块用于车队老板/客户方管理人员的网页端账号预设，不与小程序司机账号混用。

### 列表字段

- 所属客户
- 登录账号
- 账号名称
- 手机号
- 开放板块
- 状态
- 创建时间

### 操作

- 新增客户账号：选择所属客户，填写登录账号、首次密码、账号名称、手机号、状态和开放板块。
- 编辑客户账号：调整所属客户、账号名称、手机号、状态、开放板块和备注。
- 重置密码：生成新的临时密码后发给客户老板。
- 开放板块必须同步到后台专属角色菜单；客户老板登录后只能看到被开放的板块。
- 默认只开放查询和导出类权限，不默认开放新增、删除、设备控制和任务启停。

## 扫码刷卡记录

### 列表字段

- 识别结果：已识别 / 未知账户
- 操作场景：扫码校验 / 开始放电 / 刷卡识别
- 识别方式：微信扫码 / 刷卡
- 账户、手机号、卡号
- 设备、仪表、网关、电表
- 二维码内容
- 提示信息
- 发生时间

### 约束

- 每次扫码校验、开始放电和后续刷卡识别都必须写入记录。
- 未知账户也必须写入记录，方便运营人员在后台排查和补充授权。
- 充电桩显示屏联动属于后续桩端集成范围，管理端先完成记录追溯。

## 数据报表

数据报表与报表面板平级，用于面向客户输出“客户电费账单”口径，不替代报表面板中的电网费用明细。

### 页面定位

- 采用报表面板一致的筛选方式：全部电表、按场地、单个电表、账单月份。
- 数据口径以客户实际使用移动储能的放电量为主，即反向有功电能分时差值。
- 页面样式沿用报表面板的白底、青绿色标题条、紧凑表格和 A4 报表视觉。

### 展示模块

- 账单抬头：采用电费账单样式，左侧显示移动储能品牌标识，中间显示“移动储能运营管理平台 / 电费账单”，右侧显示客户服务和监督电话；下方展示账单周期、户号、户名、用电地址、用电类别、电压等级、供电服务单位、市场化属性、打印人、打印日期、报表编号和统计摘要。
- 指标卡片：本期用电量（放电）、本期平均单价、本期应付金额。
- 按时段用电：峰、平、谷、深谷的时间范围、用电量、服务单价和金额，并提供柱状图。
- 保底电量核算：约定保底电量从命中的计费规则读取，本期实际用电量、计费方式和超出/未达保底提示跟随所选范围汇总。
- 应付金额计算方式：展示 `MAX(保底电量 × 综合单价, 实际分时电量 × 各时段单价之和) + 基础服务费`，其中基础服务费由场地费、运维费、通信费、平台服务费、电池折旧成本和其他固定费用共同组成。
- 用能成本对比：自购柴油发电成本、直接接入电网估算成本、使用移动储能节省金额；柴油发电估算单价、电网接入估算下限单价和电网接入估算加价均从命中的计费规则读取。

### 导出

- 导出为固定 A4 样式 PDF。
- 手机端应优先生成 PDF 文件或打开 PDF 预览，不只依赖浏览器打印面板。
- 导出文件需记录到报表归档入口，方便后续在头像菜单或报表记录中查看。
